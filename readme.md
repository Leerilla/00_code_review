### 🚀 실습 진행 가이드: 동료의 코드 리뷰하기

이번 실습에서는 여러분이 **'리뷰어(Reviewer)'** 가 되어 동료 개발자(Bot)가 올린 코드를 리뷰합니다.
버튼 클릭 한 번으로 나만의 실습 환경을 만들고 코드 리뷰를 시작해 보세요!

#### Step 1. 실습 환경 자동 생성하기
1. 교육용 GitHub 저장소 상단의 **[Issues]** 탭을 클릭합니다.
2. 우측 상단의 초록색 **[New issue]** 버튼을 클릭합니다.
3. 목록에서 **🚀 [Chapter 1] 실습 시작하기** 우측의 `Get started` 버튼을 누릅니다.
4. 내용 수정 없이 하단의 **[Submit new issue]** 를 클릭합니다.
5. 약 10~15초 정도 기다리면, Bot이 "🎉 실습 준비가 완료되었습니다!"라는 코멘트와 함께 **실습용 PR 링크**를 달아줍니다. 링크를 클릭해 PR로 이동하세요.

#### Step 2. 리뷰어로서 피드백 남기기
PR 화면에 들어가면 우측 'Reviewers' 탭에 본인이 리뷰어로 할당되어 있는 것을 볼 수 있습니다. 이제 배운 화법을 적용해 볼 시간입니다!

1. PR 상단의 **[Files changed]** 탭을 클릭하여 코드를 확인합니다.
2. `DiscountCalculator.java` 파일에서 문제가 되거나 칭찬할 만한 코드 라인에 마우스를 올려 파란색 **`+` 버튼**을 클릭합니다.
3. ✍️ *예시: "이 부분 로직이 깔끔하네요! 다만 추가 할인율에 대한 경계값 검증이 누락된 것 같은데, 우리가 상단에 예외 처리를 추가하면 어떨까요?"*
4. 코멘트 작성 후 단일 코멘트가 아닌, **[Start a review]** 녹색 버튼을 클릭합니다.
5. 다른 리뷰 포인트(논리적 결함, 성능 문제 등)도 찾아 추가로 코멘트를 남긴 뒤 **[Add review comment]** 를 클릭합니다.
6. 우측 상단의 **[Review changes]** 를 누르고, 총평을 적은 뒤 **[Submit review]** 를 클릭하여 리뷰를 완료합니다.

> 🤖 **[AI 리뷰 가이드 확인하기]** <br>
> `Submit review`를 누르면 백그라운드에서 대기 중이던 AI 강사가 여러분의 리뷰 화법과 내용을 분석합니다! 잠시 후 PR 페이지를 새로고침 하면, 내가 남긴 리뷰가 적절했는지 AI가 꼼꼼하게 평가해 준 피드백을 확인할 수 있습니다.
